My Projects

DHCP STARVATION ATTACK

The project aims to execute a "DHCP Starvation" attack followed by the introduction of a rogue DHCP server in a local network. The goal is to demonstrate how an attacker can deplete the available IP addresses on a legitimate DHCP server and then deploy a malicious DHCP server to control network configurations of clients. Step 1: DHCP Starvation Use a tool like "Yersinia" or "dhcpstarv" to flood the DHCP server with IP lease requests until all available addresses are exhausted. Observe the network impact and the difficulties new clients face in obtaining an IP address Step 2: Rogue DHCP Set up a rogue DHCP server using a tool like "Rogue DHCP Server" to respond to clients' DHCP requests. Configure the rogue DHCP server to redirect clients' traffic to malicious DNS servers or network gateways controlled by the attacker.. .

PENTEST LINUX EXEGOLE MACHCINE

The project involves simulating a penetration test on a Linux machine using Exegol, a popular platform for red team exercises. Exegol is a pre-configured virtual machine loaded with numerous tools for penetration testing and advanced attack simulations. Step 1: Setting up the Exegol Machine Download and configure the Exegol virtual machine. Ensure essential tools like Metasploit, Nmap, and other post-exploitation tools are installed and updated. Step 2: Reconnaissance and Scanning Use reconnaissance tools to identify services and vulnerabilities on the target machine. Perform scans with Nmap to detect open ports and running services. Step 3: Exploitation Use Metasploit to exploit discovered vulnerabilities. Deploy payloads to gain shell access on the target machine. Step 4: Post-Exploitation Escalate privileges to gain administrative rights on the machine. Extract sensitive data and create backdoors for future access
